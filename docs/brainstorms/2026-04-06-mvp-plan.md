# FieldPulse Prompt Library — MVP Plan

**Date:** 2026-04-06
**Author:** Hamza Saraswat
**Stack:** React + Vite + Tailwind (frontend) / Supabase (backend) / Vercel (hosting)
**Goal:** Take the static prototype to a functional product that team leads can use and give feedback on.

---

## What the MVP Delivers

A working prompt library where employees can browse, search, copy, and comment on prompts — backed by a real database with authentication. Admins get a full dashboard to manage all content. Users can submit new prompts for review.

---

## 1. Backend & Infrastructure

### 1.1 Database (Supabase / Postgres)

All data moves from hardcoded JSON files to Supabase. Here are the tables we need:

| Table | What it stores |
|---|---|
| **profiles** | User info — display name, email, role (user/admin), department. Auto-created on first login via a database trigger extending Supabase `auth.users`. |
| **prompts** | The core content — title, overview, prompt text, category, author, version, trending flag, timestamps. |
| **tags** | Admin-managed tag list with a type field ("department" or "workflow") for color coding. |
| **prompt_tags** | Many-to-many join linking prompts to tags. |
| **variables** | Template variables per prompt — name, description, display order. |
| **categories** | Use-case categories (Outreach, Onboarding, Debugging, etc.) that organize the sidebar. |
| **comments** | Text feedback on prompts — linked to author and prompt, timestamped. |
| **favorites** | User-prompt pairs powering the favorites feature. |
| **bundles** | Named groups of related prompts (e.g., "New Customer Onboarding"). |
| **bundle_prompts** | Join table linking bundles to prompts with sort order. |
| **version_history** | Snapshot of each prompt version — full text, change description, author, timestamp. |

All tables use **Row Level Security (RLS)** policies to enforce permissions based on user role.

### 1.2 Authentication

**Google OAuth via Supabase Auth.** One-click "Sign in with Google" — all FieldPulse employees have Google Workspace accounts. Supabase handles the OAuth flow, session management, and token refresh. No passwords to manage.

### 1.3 Roles

Two roles, kept simple:

| Role | What they can do |
|---|---|
| **User** | Browse, search, copy prompts. Favorite and comment. Submit new prompts (pending admin approval). |
| **Admin** | Everything a user can do, plus: create/edit/delete prompts directly, approve or reject submissions, manage tags and categories, manage bundles, assign user roles. |

### 1.4 Hosting & Deployment

- **Vercel** hosts the React app. Auto-deploys on push to `main`.
- **Supabase** hosts the database, auth, and any edge functions.
- Environment variables (Supabase URL + anon key) configured in Vercel dashboard.

### 1.5 Data Migration

Migrate the 22 existing prompts (currently hardcoded in `prompts.ts`) into Supabase. This includes all tags, variables, categories, bundles, and version history. One-time seed script.

## 2. Admin Dashboard

A dedicated admin area accessible from the sidebar (visible only to admin users). This is where all content management happens.

### 2.1 Prompt Management

- **All prompts table** — Full list of every prompt with title, author, category, tags, status, and last updated date. Click to edit.
- **Prompt editor** — Create or edit a prompt: title, overview, prompt text, variables, tags, category, bundle assignment. Preview how it looks before saving.
- **Delete / archive** — Remove prompts with confirmation.

### 2.2 Submission Queue

- **Pending submissions** — List of prompts submitted by users awaiting review. Shows title, author, submitted date.
- **Review flow** — Admin can preview the full prompt, leave feedback comments, then approve (goes live) or reject (with reason sent back to the submitter).

### 2.3 Tag & Category Management

- **Tags** — Add, rename, or remove tags. Set tag type (department vs workflow) which controls the blue/purple color coding.
- **Categories** — Add, rename, or reorder the use-case categories that populate the sidebar navigation.

### 2.4 Bundle Management

- **Create / edit bundles** — Name, description, and select which prompts belong to the bundle with drag-and-drop ordering.
- **Preview** — See how the bundle renders in the user-facing card view.

### 2.5 User Management

- **User list** — All authenticated users with their role and last active date.
- **Role assignment** — Promote a user to admin or demote back to user.

## 3. User-Facing Features

### 3.1 What Carries Over from the Prototype (Already Built)

These features are functional in the current prototype and will be connected to the Supabase backend:

- Card and table views with toggle (card default)
- Tag-based filtering with color-coded badges (blue = department, purple = workflow)
- Search across title, overview, prompt text, and tags
- Sortable table (title A-Z / Z-A)
- Prompt detail pages with shareable URLs (`/prompts/:id`)
- Prompt drawer (slide-out quick preview)
- Split copy button with "Open in ChatGPT / Claude / Gemini" dropdown
- Variable fill-in system with live preview
- Collapsible "See it in Action" use case examples
- Comments section (currently local state — will be persisted)
- Favorites (currently localStorage — will be persisted)
- FieldPulse branding with "AI for the Field" tagline
- Dark mode
- Sidebar navigation by use-case groups
- Category pills for sub-filtering
- Bundles displayed as grouped cards

### 3.2 New for MVP

**Copy button on prompt cards**
The split copy button (Copy + AI platform dropdown) returns to the card footer. Users can copy a prompt directly from the card view without opening the drawer. The card click still opens the drawer for full details.

**Real AI platform logos**
Replace the colored dots in the "Open in" dropdown with actual ChatGPT, Claude, and Gemini SVG icons.

**Functional AI platform URLs**
When a user clicks "Open in ChatGPT," the prompt is copied to clipboard AND the platform opens with the prompt pre-filled where supported:

| Platform | Behavior |
|---|---|
| **ChatGPT** | Opens `chatgpt.com/?q={prompt}` — pre-fills the input box |
| **Claude** | Opens `claude.ai/new` — prompt copied to clipboard (URL pre-fill may not be supported) |
| **Gemini** | Opens `gemini.google.com` — prompt copied to clipboard (no native URL pre-fill) |

All three always copy to clipboard first as a safety net.

**Prompt submission form**
Any logged-in user can submit a new prompt via a structured form (title, prompt text, overview, variables, tags, category). Submitted prompts go to the admin review queue and aren't visible to other users until approved.

**Persistent comments**
Comments save to the database and persist across sessions. Users see all comments from all users on each prompt.

**Persistent favorites**
Favorites sync to the database per user — accessible from any device after login.
